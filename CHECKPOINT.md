# Чекпойнт (самари контекста)

**Дата:** апрель 2026 · Репозиторий vacation planner (frontend React+Vite, backend FastAPI, PostgreSQL).

---

## Локальный запуск

1. **Docker Desktop** запущен.
2. Если CLI не видит Docker: `docker context use desktop-linux`.
3. Из корня: **`docker compose --profile dev up -d --build`** — откройте **`http://localhost:5173`**.  
   Либо **`cp .env.example .env`** и **`docker compose up -d --build`** (в шаблоне уже **`COMPOSE_PROFILES=dev`**).

Демоданные один раз: **`docker compose exec backend python seed.py`**.

### Важно

- На Windows для hot reload в Vite включён **`server.watch.usePolling`** в `frontend/vite.config.ts`.
- Сервисы **`backend`** и **`frontend`** (Vite) идут с профилем **`dev`**. Команда **`docker compose --profile dev up -d`** работает без настройки **`COMPOSE_PROFILES`** в `.env`. Прод — **`backend_prod`** + **`frontend_prod`**, профиль **`prod`**.
- Если **`5173`** недоступен: выполните **`docker compose ps`** и **`docker compose logs frontend`**.

## Деплой (Dokploy)

- Один **`docker-compose.yml`**. Для прода задайте **`COMPOSE_PROFILES=prod`** — поднимутся **`frontend_prod`** (nginx) и **`backend_prod`**.
- Задайте **`POSTGRES_PASSWORD`** (и при необходимости **`POSTGRES_USER`**).
- Локально **`5173`** — см. п.3 ( **`--profile dev`** или **`.env` с `COMPOSE_PROFILES=dev`** ).

## Сотрудник: «Мой отпуск» / «Мои пожелания» (логика UI)

- **Баланс:** «Запланировано» и остаток считаются по **сохранённым пожеланиям** (приоритет варианта v1 → v2 → v3 с заполненными датами).
- **`frontend/src/lib/utils.ts`:** для строк **`YYYY-MM-DD`** даты и **`daysBetween`** считаются как календарные (без UTC-сдвига), чтобы цифры совпадали с полями ввода.
- Блок **«Запланированный отпуск»:** если есть сохранённые пожелания — показываются **все заполненные варианты v1–v3** в таблице (не черновик `vacation_block` из системы); статус справа: *«Статус: ожидание согласования менеджером»*. В балансе сверху «Запланировано» по-прежнему по приоритету **v1 → v2 → v3**.
- Страница **«Мои пожелания»:** три варианта описаны как альтернативы одного согласованного периода; лимит длины варианта — остаток дней в системе; цвет «Доступно» от остатка ≤ 0, не от числа 14.
- Блок **«Назначенный отпуск»** — только при статусе блока **`APPROVED`**.
- Поле «По варианту» у блоков отпуска убрано как лишнее.
- Превышение лимита по дням: предупреждение перенесено на страницу **«Мои пожелания»** (оранжевый текст у варианта); с баланса убрано.

## Полезные команды Git

Точки для отката искать по сообщениям коммитов, например:

- `Деплой сделан. Начинаем накручивать крассоту` — после фикса compose без порта 5432.
- `fix(dev): Vite watch polling for Docker Desktop on Windows bind mounts`
- `fix(employee): Запланированный отпуск — приоритет сохранённых пожеланий над черновиком блока`
- последующие `ui(employee): …`, `fix:` календарные даты и т.д.
